
# ScoreFormat.md（B.Click 保存データ形式 — AI向け仕様）

B.Click の保存データ（PDFに添付される *.json）の形式を説明します。
AIにリズム譜データを生成してもらうときは、このファイルを参照してください。

**ソースの正**: `www/releases/next/assets/js/utils/scoreSerialization.js` の `##Spec` コメント
保存形式を変更したときは、このファイルも更新すること。

---

## JSON 全体構造

```json
{
  "schemaVersion": 0,
  "score": {
    "tempo": 80,
    "clickCount": 4,
    "countIn": 4,
    "timeSignature": "4/4",
    "measures": 4,
    "progression": "Am F C G",
    "barsPerRow": 2,
    "scoreEnabled": true,
    "clickTonePattern": ["A5", "A4", "A4", "A4"],
    "rhythmPattern": ["4", "4", "4", "4"],
    "bars": [ ...後述... ]
  }
}
```

### キーの説明

| キー               | 説明                                              | 制約                              |
|--------------------|---------------------------------------------------|-----------------------------------|
| `schemaVersion`    | 保存形式のバージョン（固定値）                    | 必ず `0`                          |
| `tempo`            | BPM テンポ                                        | 30〜240                           |
| `clickCount`       | 拍数（クリック音の数）                            | 1〜16                             |
| `countIn`          | カウントイン（拍数）                              | 0〜10                             |
| `timeSignature`    | 拍子                                              | `"2/4"` `"3/4"` `"4/4"` のいずれか |
| `measures`         | 小節数                                            | 1〜50                             |
| `progression`      | コード進行（スペース区切り）                      | 最大10コード、空文字 `""` も可    |
| `barsPerRow`       | 1段あたりの小節数                                 | 1〜4                              |
| `scoreEnabled`     | リズム表示のON/OFF                                | `true` または `false`             |
| `clickTonePattern` | 拍ごとのクリック音色配列                          | `"A5"` `"A4"` `""` のみ、長さ = `clickCount` |
| `rhythmPattern`    | 全小節共通のデフォルトリズムパターン              | 音価の合計が拍数と一致すること    |
| `bars`             | 小節データの配列                                  | 長さ = `measures`                 |

---

## 音価トークン（rhythm に使える値）

| トークン | 音価       | 拍数換算 |
|----------|------------|----------|
| `"1"`    | 全音符     | 4拍      |
| `"2"`    | 2分音符    | 2拍      |
| `"4"`    | 4分音符    | 1拍      |
| `"8"`    | 8分音符    | 0.5拍    |
| `"16"`   | 16分音符   | 0.25拍   |
| `"r1"` `"r2"` `"r4"` `"r8"` `"r16"` | 各種休符 | 上記と同じ |
| `"t4"` `"t8"` など | タイ付き音符 | 上記と同じ |

**ルール**: rhythm 配列の音価合計が、拍子の拍数（例: 4/4 なら 4）と一致しなければならない。

## 小節（bar）の構造

```json
{
  "chord":  [ ["Am","","",""], ["","","",""], ["C","","",""], ["","","",""] ],
  "rhythm": [ "4", "4", "4", "4" ],
  "lyrics": [ ["よ","","",""], ["る","","",""], ["の","","",""], ["そ","","",""] ]
}
```

### chord / lyrics の構造

- 2次元配列: `拍数 × 4（サブ分割数）`
- 4/4拍子なら `chord.length === 4`、各要素の長さも `4`
- `chord[拍番号][0]` = その拍のコード名（例: `"Am"`）
- `chord[拍番号][1]` = 1拍内で2つ目のコード（8分音符2つなど）
- `chord[拍番号][2]〜[3]` = さらなる細分（通常は `""` でよい）
- `lyrics` も chord と同じ構造。`lyrics[拍番号][サブ]` に歌詞の1文字を入れる
- `lyrics` は省略可能（省略した場合は空文字で補完される）

### 8分音符の歌詞配置例（4/4、1小節）

rhythm: `["8", "8", "4", "4", "4"]` の場合（1拍目に8分音符×2、2〜4拍目に4分音符）

```json
{
  "chord":  [ ["Am","C","",""], ["F","","",""], ["G","","",""], ["Am","","",""] ],
  "rhythm": [ "8", "8", "4", "4", "4" ],
  "lyrics": [ ["く","ら","",""], ["い","","",""], ["よ","","",""], ["る","","",""] ]
}
```

```

- 小節数：4
- BPM：80
- 拍子：4/4
- 使用する音符：4分音符と8分音符
- コード進行：（自由 / または指定）
- 歌詞テーマ：（例：夜）

JSON のみ出力してください。
```
